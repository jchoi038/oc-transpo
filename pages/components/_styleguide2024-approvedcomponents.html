Icon naming convention guide

Format:

 
                    [A]
                    _
                    [B]
                    .svg
[A]

Category Name:

  • Pick a category from the following options: 'line' / 'route' / 'transport' / 'semantic' / 'accessibility' / 'station' / 'resource' / 'destination' / 'interactive' / 'contact' / 'detour' / 'map' / 'tags' / 'service' / 'msc' / 'link-icon'
  • Use lowercase letters only.
    • [B]

    Icon Name:

  • Name the icon using kebab-case format.
  • Correct: happy-goat
  • Incorrect: happyGoat, happy_goat
  • Use lowercase letters only.
    • _

    Linking [A] and [B]

  • Combine the category name and icon name with an underscore _.
  • Example:'contact_email.svg'
    • .svg

    Upload format

  • upload your icon in '.svg' format

    • Upload directory

    All icons are to be saved in 

    Assets/Uploads/icons

    Usage guide

    Use case scenarios

    The promotional banner component is designed to drive user engagement by directing users to actions such as learning more, signing up, or downloading resources (apps, pdfs, images etc).

    Images & icons

    • Incorporate relevant images or icons to enhance the message or service logos
    • Provide alt text for images

    Content:

    1. Banner title
      • Best practice: make the title of the banner action-oriented. Ex: “Sign up for My Alerts”, “Stay up to date”, “Contact us!” etc
      • Banner titles should be short and to the point, avoiding wordiness to maintain a consistent height and accommodate mobile display
    2. Banner body text
      • Keep messages clear and action-oriented to drive engagement.
      • Limit the banner description text to a maximum of 1-2 sentences.
    3. CTA
      • CTA placement: in the case of two CTA buttons, the primary CTA (red border) is placed on the right-hand side, and the secondary CTA (grey) is placed on the left
      • The maximum number of CTA buttons is 2. Place additional links in the body if necessary
      • Use actionable phrases such as “Learn More,” “Explore Services,” or “Get Updates”
      • Avoid making the text too long and instead hide more descriptive text using the class="visuallyhidden"
      • Provide a helpful title attribute to the buttons

    Responsive behaviour (<576 px)

    1. Buttons: The banner button(s) will be hidden from view and the entire banner will be clickable with the banner inheriting the link (href property) from the button.
      • If banner has more than one CTA: The link of the last CTA will be used for the mobile view of the banner.
      • To prevent buttons from hiding from view: add class .prevent-default
    2. Text: Text will decrease in size to accomodate smaller viewport widths
    3. Icon/image: icon / image will decrease in size to accomodate smaller viewport widths
